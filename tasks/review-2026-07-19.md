## T8 — `CURRENT_CONTEXT` RAII guard  *(R10)*

**Severity:** Med (raw-pointer soundness gap).
**Verify:** `context/mod.rs:82` `reject_eval_var_lexical_conflict(&program,
ctx)?` early-returns before the restore at `context/mod.rs:104`. On
this `Err` path `CURRENT_CONTEXT` keeps the new `ctx_ptr` instead of
the outer `prev_ctx`, leaving a dangling pointer if the outer context
was reset or dropped by the caller.

- [ ] **Repro test:** a `#[test]` that runs a long `eval` chained three
      deep, drops the middle `Context`, evaluates in a new `Context`,
      observes the stale `CURRENT_CONTEXT` pointer (detectable by
      Wrapping a sentinel global). It fails today.
- [ ] `struct CtxGuard { prev: Option<*mut Context> } impl Drop {
      ... restore prev }`; `eval_impl` constructs one at the top.
- [ ] Replace `Cell<Option<Value>>::take` + `set` peek pattern (≈150
      use sites) with `RefCell<Option<Value>>` + `.borrow().clone()`.
- [ ] R6 moves per-realm pointers onto `Realm`; `reset` becomes
      trivial.

## T14 — Drop `FROZEN_OBJECTS` thread_local address-keyed set  *(new R16)*

**Severity:** Med (soundness, not just LOC).
**Verify:** `builtins/object_static.rs:13-22` stores `Rc::as_ptr(o) as
usize` in a `thread_local Vec<usize>` on `Object.freeze`. Never cleared
on `Context::reset`. Once the `Rc<Object>` is dropped the address can
be reused by a later allocation; `is_frozen_object` then reports the
new object as frozen. Also leaks monotonically across `reset`.

- [ ] **Repro test:** `#[test]` freezing an object, dropping the
      context (`reset`), allocating a new object that happens to
      reuse the address (deterministic via a fixed global allocation
      pattern in the test) and asserting `Object.isFrozen(o) === false`.
      Construct a deterministic variant by allocating in a loop until
      the previous address reappears, then check `is_frozen_object`.
- [ ] Use the existing `Object.extensible` boolean field (already
      present). `Object.freeze` flips it to false; `is_frozen_object`
      reads `!o.extensible` (plus `Object.isSealed` semantics if
      needed, but NOT via a global pointer set).
- [ ] Delete `FROZEN_OBJECTS` thread_local + `is_frozen_object` helper.
- [ ] Acceptance: repro test flips red→green; `rg 'FROZEN_OBJECTS'
      crates/quench-runtime/src` returns zero hits.
