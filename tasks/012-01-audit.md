## Known Parity Differences

### 1. ink-use-effect
- **Deno:** `Component mounted: Yes`, `Count: 0`
- **HIR:** `Component mounted: No`, `Count:` (empty)
- **Reason:** useEffect not running before first render in HIR

### 2. Interactive examples (useInput)
- **Deno:** Shows live updating values
- **HIR:** Shows static initial state only
- **Reason:** Input handling not implemented in HIR runtime

## Recommendations

1. **For HIR Runtime:** Document hook limitations clearly
2. **For Parity Tests:** Accept static output as valid for HIR
3. **For useEffect:** Consider running effects before first render
